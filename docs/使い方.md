## check_qnn.py - QNN環境確認

### 概要

ONNX Runtime QNN Execution Providerが正しくインストールされ、利用可能かを確認するスクリプトです。

### 使い方

```powershell
python check_qnn.py
```

### 出力の見方

#### ✅ 成功時

```
============================================================
ONNX Runtime QNN Status Check
============================================================

📦 ONNX Runtime Location:
   C:\Users\...\site-packages\onnxruntime\__init__.py

🔌 Available Execution Providers:
   🚀 QNNExecutionProvider          ← QNNが利用可能
      AzureExecutionProvider
      CPUExecutionProvider

✅ SUCCESS: QNN Execution Provider is available!
   Your NPU hardware acceleration is ready to use.

🧪 Testing QNN Session Creation...
Starting stage: Graph Preparation Initializing
...
   Active Providers: ['QNNExecutionProvider', 'CPUExecutionProvider']
   ✅ QNN session created successfully!

============================================================
```

**確認ポイント:**
- 🚀 マークが `QNNExecutionProvider` に付いている
- "SUCCESS" メッセージが表示される
- セッション作成が成功する

#### ❌ 失敗時

```
🔌 Available Execution Providers:
      CPUExecutionProvider            ← QNNが無い

❌ WARNING: QNN Execution Provider is NOT available.
   Inference will fall back to CPU.
```

**対応:**
1. `pip install onnxruntime-qnn` を実行
2. NPU搭載デバイスか確認

### いつ実行するか

- 初回セットアップ後
- `simple_demo.py`でCPUモードになった時
- `onnxruntime-qnn`を再インストールした後

## simple_demo.py - 物体検知デモ

### 概要

YOLOv8モデルを使用したリアルタイム物体検知デモです。4種類の入力ソースに対応しています。

### 基本的な使い方

```powershell
python simple_demo.py
```

### 起動フロー

1. **環境情報の表示**
   ```
   ============================================================
   Simple NPU Detection Demo (Direct ORT)
   ============================================================
   ORT Location: C:\...\onnxruntime\__init__.py
   Available Providers: ['QNNExecutionProvider', 'CPUExecutionProvider']
   ```

2. **モデルの読み込み**
   ```
   📦 モデルを読み込み中...
   ✅ セッション作成成功
   Active Providers: ['QNNExecutionProvider', 'CPUExecutionProvider']
   🚀 QNN (NPU) が有効です！
   ```

3. **入力ソースの選択**
   ```
   入力ソースを選択してください:
   1. Webカメラ (デフォルト)
   2. 画像ファイル
   3. 動画ファイル
   4. RTSPストリーム
   
   選択 (1-4, Enter=1):
   ```

4. **推論実行**

   リアルタイムで物体検知が実行され、結果が画面に表示されます。

### 操作方法

| キー | 動作 |
|------|------|
| `q` | プログラムを終了 |

### 画面表示

```
┌─────────────────────────────────────┐
│ FPS: 28.5 (Inf: 12.3ms)             │ ← パフォーマンス情報
│ Device: NPU (QNN)                   │ ← 使用デバイス
│                                     │
│    ┌──────────────┐                 │
│    │ person 0.92  │                 │ ← 検知結果（クラス 信頼度）
│    └──────────────┘                 │
└─────────────────────────────────────┘
```

#### 表示項目の説明

- **FPS (Frames Per Second)**: 1秒あたりの処理フレーム数
  - NPU使用時: 28-30 FPS
  - CPU使用時: 8-10 FPS

- **Inf (Inference time)**: 1フレームの推論時間（ミリ秒）
  - NPU使用時: 12-15 ms
  - CPU使用時: 100-120 ms

- **Device**: 使用しているアクセラレータ
  - `NPU (QNN)`: NPUで実行中 ✅
  - `CPU`: CPUで実行中（フォールバック）

- **検知結果**: 緑の枠とラベル
  - 枠: 検知された物体の位置
  - ラベル: `クラス名 信頼度`

## 入力ソース別の使い方

### 1. Webカメラ

**選択:**
```
選択 (1-4, Enter=1): 1
```
または単にEnterキー

**特徴:**
- 最もシンプル
- PCの内蔵カメラまたは外付けUSBカメラを使用
- リアルタイム処理

**トラブルシューティング:**
- カメラが開けない → 他のアプリがカメラを使用していないか確認
- 映像が表示されない → デバイスマネージャーでカメラを確認

### 2. 画像ファイル

**選択:**
```
選択 (1-4, Enter=1): 2
画像ファイルのパスを入力: C:\path\to\image.jpg
```

**特徴:**
- 静止画の推論
- 推論時間を正確に計測可能
- 結果を確認後、任意のキーで終了

**対応形式:**
- JPEG (.jpg, .jpeg)
- PNG (.png)
- BMP (.bmp)
- その他OpenCVが対応する画像形式

**実行例:**
```powershell
python simple_demo.py
# 2 を入力
# C:\Users\username\Pictures\test.jpg を入力
```

**出力例:**
```
推論時間: 12.34 ms
```

結果画像が表示され、検知された物体が緑の枠で囲まれます。

### 3. 動画ファイル

**選択:**
```
選択 (1-4, Enter=1): 3
動画ファイルのパスを入力: C:\path\to\video.mp4
```

**特徴:**
- 動画ファイルをフレームごとに処理
- 動画の最後まで行くと自動的に最初に戻る（ループ再生）
- `q`キーで終了

**対応形式:**
- MP4 (.mp4)
- AVI (.avi)
- その他OpenCVが対応する動画形式

**実行例:**
```powershell
python simple_demo.py
# 3 を入力
# C:\Users\username\Videos\test.mp4 を入力
```

### 4. RTSPストリーム

**事前準備:**

プロジェクトルートに `.env` ファイルを作成：

```env
RTSP_URL=rtsp://192.168.1.100:554/stream1
RTSP_USERNAME=admin
RTSP_PASSWORD=yourpassword
```

**選択:**
```
選択 (1-4, Enter=1): 4
```

`.env`の情報を自動的に読み込んで接続します。

**特徴:**
- ネットワークカメラからリアルタイム映像取得
- 認証情報は自動的にURLエンコード（特殊文字対応）
- 接続エラー時は適切なエラーメッセージを表示

**RTSP URL形式:**

- 基本形式: `rtsp://IPアドレス:ポート/パス`
- 認証付き: `rtsp://ユーザー名:パスワード@IPアドレス:ポート/パス`

**例:**
```env
# 認証なし
RTSP_URL=rtsp://192.168.1.100:554/stream1

# 認証あり（.envで分割）
RTSP_URL=rtsp://192.168.1.100:554/stream1
RTSP_USERNAME=admin
RTSP_PASSWORD=12345

# 認証あり（URL内指定も可）
RTSP_URL=rtsp://admin:12345@192.168.1.100:554/stream1
```

**注意:**
- パスワードに `:` や `@` などの特殊文字が含まれる場合は、`.env`で分割指定を推奨
- プログラムが自動的にURLエンコードするため、特殊文字も正しく処理されます

**トラブルシューティング:**

| 症状 | 原因 | 対策 |
|------|------|------|
| 接続できない | URLが間違っている | `.env`のRTSP_URLを確認 |
| 認証エラー | ユーザー名/パスワードが間違っている | `.env`の認証情報を確認 |
| ネットワークエラー | カメラがオフラインまたはファイアウォール | ping、ネットワーク設定を確認 |
| ポートエラー | ポート番号が間違っている | カメラの設定を確認（通常554） |

## カスタマイズ

### 検知パラメータの調整

`simple_demo.py` の以下の部分を編集することで、検知感度を調整できます：

```python
# 93行目付近
def postprocess(output, scale, padding, conf_threshold=0.5, iou_threshold=0.45):
```

**パラメータ:**

- `conf_threshold` (信頼度閾値): 0.0 ~ 1.0
  - **デフォルト:** 0.5
  - **小さくする:** より多くの物体を検知（誤検知も増える）
  - **大きくする:** 確実な検知のみ（検知漏れが増える）
  - **推奨値:** 0.3 ~ 0.7

- `iou_threshold` (NMS閾値): 0.0 ~ 1.0
  - **デフォルト:** 0.45
  - **小さくする:** 重複する検知を除去しやすい
  - **大きくする:** 重複する検知を許容
  - **推奨値:** 0.3 ~ 0.6

**例: より多く検知したい場合**
```python
def postprocess(output, scale, padding, conf_threshold=0.3, iou_threshold=0.45):
```

**例: 確実な検知のみ**
```python
def postprocess(output, scale, padding, conf_threshold=0.7, iou_threshold=0.3):
```

### 入力サイズの変更

YOLOv8のデフォルト入力サイズは 640x640 です。変更する場合：

```python
# 29行目
def preprocess(frame, input_size=(640, 640)):
```

**例: 高解像度（精度向上、速度低下）**
```python
def preprocess(frame, input_size=(1280, 1280)):
```

**例: 低解像度（速度向上、精度低下）**
```python
def preprocess(frame, input_size=(320, 320)):
```

**注意:** モデルが対応するサイズに限定されます。

### QNNオプションの調整

`simple_demo.py` の159-164行目:

```python
qnn_options = {
    'backend_path': 'QnnHtp.dll',           # バックエンドDLL
    'qnn_context_priority': 'high',         # 優先度: low, normal, high
    'profiling_level': 'basic'              # プロファイリング: off, basic, detailed
}
```

**オプション解説:**

- `backend_path`: QNNバックエンド
  - `QnnHtp.dll`: Hexagon Tensor Processor（NPU）
  - `QnnCpu.dll`: CPU（デバッグ用）

- `qnn_context_priority`: 実行優先度
  - `low`: 省電力
  - `normal`: バランス
  - `high`: 高性能

- `profiling_level`: プロファイリングレベル
  - `off`: 無効（最速）
  - `basic`: 基本情報
  - `detailed`: 詳細情報（デバッグ用）

### 検知クラスのフィルタリング

特定のクラスのみ検知したい場合、`postprocess`関数を修正：

```python
# 検知したいクラスID（personのみ検知する例）
TARGET_CLASSES = [0]  # 0 = person

def postprocess(output, scale, padding, conf_threshold=0.5, iou_threshold=0.45):
    predictions = np.squeeze(output[0]).T
    boxes = predictions[:, :4]
    scores = predictions[:, 4:]
    
    class_ids = np.argmax(scores, axis=1)
    confidences = scores[np.arange(len(class_ids)), class_ids]
    
    # --- ここに追加 ---
    class_mask = np.isin(class_ids, TARGET_CLASSES)
    boxes = boxes[class_mask]
    confidences = confidences[class_mask]
    class_ids = class_ids[class_mask]
    # --- ここまで ---
    
    mask = confidences > conf_threshold
    # 以下省略
```

**クラスID一覧:**
- 0: person
- 2: car
- 15: cat
- 16: dog
- その他は `COCO_CLASSES` リスト参照（16-27行目）

## 💡 Tips

### パフォーマンスを向上させる

1. **QNNの優先度を上げる**
   ```python
   qnn_options = {
       'qnn_context_priority': 'high',  # low → high
   }
   ```

2. **入力サイズを小さくする**
   ```python
   input_size=(320, 320)  # 640 → 320
   ```

3. **検知閾値を上げる**
   ```python
   conf_threshold=0.7  # 0.5 → 0.7
   ```

### 検知精度を向上させる

1. **入力サイズを大きくする**
   ```python
   input_size=(1280, 1280)
   ```

2. **検知閾値を下げる**
   ```python
   conf_threshold=0.3
   ```

3. **照明条件を改善する**
   - 明るい環境で撮影
   - 逆光を避ける

## ❓ よくある質問

**Q: 検知結果を保存したい**

A: 現在のバージョンでは画面表示のみです。保存が必要な場合は、`cv2.imwrite()` を使用してカスタマイズしてください。

**Q: 複数のカメラを同時に使いたい**

A: 現在のバージョンでは1つの入力のみ対応。複数カメラが必要な場合は、マルチプロセスでの実装が必要です。

**Q: 独自のYOLOv8モデルを使いたい**

A: `models/yolov8n.onnx` を独自モデルに置き換えることで可能です。入力・出力形式が同じであることを確認してください。

**Q: NPUとCPUで精度は変わる？**

A: 変わりません。推論速度のみ変わります。

**Q: バッテリー持続時間は？**

A: NPU使用時は消費電力が低いため、CPU使用時よりもバッテリー持続時間が長くなります。
