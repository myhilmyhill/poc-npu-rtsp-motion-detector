# クイックスタートガイド

NPU Object Detection Demoを最短で動作させるためのガイドです。

## 📋 前提条件

- Windows 10/11
- Python 3.10以上
- Qualcomm NPU搭載デバイス（Snapdragon X Elite/Plusなど）
- インターネット接続（初回セットアップ時）

## 🚀 5分でスタート

### ステップ1: 環境確認

PowerShellを開いて、Pythonバージョンを確認：

```powershell
python --version
```

**出力例:** `Python 3.10.x` または `3.11.x`

### ステップ2: プロジェクトのクローン

```powershell
git clone <repository-url>
cd poc-npu-rtsp-motion-detector
```

### ステップ3: 依存パッケージのインストール

```powershell
pip install onnxruntime-qnn opencv-python numpy python-dotenv
```

**注意:** `onnxruntime-qnn`は通常の`onnxruntime`とは別のパッケージです。

### ステップ4: QNN環境の確認

```powershell
python check_qnn.py
```

**成功例:**
```
============================================================
ONNX Runtime QNN Status Check
============================================================

📦 ONNX Runtime Location:
   C:\...\onnxruntime\__init__.py

🔌 Available Execution Providers:
   🚀 QNNExecutionProvider
      CPUExecutionProvider

✅ SUCCESS: QNN Execution Provider is available!
   Your NPU hardware acceleration is ready to use.

🧪 Testing QNN Session Creation...
   Active Providers: ['QNNExecutionProvider', 'CPUExecutionProvider']
   ✅ QNN session created successfully!
```

✅ が表示されればOKです！

❌ が表示された場合は、[トラブルシューティング](#トラブルシューティング)を参照してください。

### ステップ5: デモの実行

```powershell
python simple_demo.py
```

起動後、入力ソースを選択：

```
入力ソースを選択してください:
1. Webカメラ (デフォルト)
2. 画像ファイル
3. 動画ファイル
4. RTSPストリーム

選択 (1-4, Enter=1):
```

**初回は `1` (Webカメラ)を選択**してEnterキーを押してください。

### ステップ6: 動作確認

- カメラ映像が表示される
- 物体（人、物など）が検知されると緑の枠が表示される
- 画面左上に「Device: NPU (QNN)」と表示されていることを確認

✅ **おめでとうございます！NPUで物体検知が動作しています！**

### 終了方法

- ウィンドウをアクティブにして `q` キーを押す

## 📸 入力ソースの使い方

### 1. Webカメラ

最もシンプルな方法。PC内蔵カメラまたは外付けUSBカメラを使用。

```
選択 (1-4, Enter=1): 1
```

### 2. 画像ファイル

静止画で推論を試したい場合。

```
選択 (1-4, Enter=1): 2
画像ファイルのパスを入力: C:\path\to\image.jpg
```

推論結果が表示されます（任意のキーで終了）。

### 3. 動画ファイル

動画ファイルをループ再生しながら推論。

```
選択 (1-4, Enter=1): 3
動画ファイルのパスを入力: C:\path\to\video.mp4
```

### 4. RTSPストリーム

ネットワークカメラから映像を取得。事前に `.env` ファイルの設定が必要です。

#### .env ファイルの作成

プロジェクトルートに `.env` ファイルを作成：

```env
RTSP_URL=rtsp://192.168.1.100:554/stream1
RTSP_USERNAME=admin
RTSP_PASSWORD=yourpassword
```

または `.env.example` をコピー：

```powershell
copy .env.example .env
notepad .env
```

#### RTSPで実行

```
選択 (1-4, Enter=1): 4
```

自動的に `.env` からURL・認証情報を読み込んで接続します。

## 📊 画面の見方

デモ実行中の画面には以下が表示されます：

```
┌─────────────────────────────────────┐
│ FPS: 28.5 (Inf: 12.3ms)             │  ← フレームレート・推論時間
│ Device: NPU (QNN)                   │  ← 使用デバイス
│                                     │
│    ┌──────────────┐                 │
│    │ person 0.92  │                 │  ← 検知結果（クラス名・信頼度）
│    └──────────────┘                 │
│                                     │
│           ┌─────────┐               │
│           │ cat 0.85│               │
│           └─────────┘               │
└─────────────────────────────────────┘
```

- **FPS**: 1秒あたりのフレーム数（高いほど滑らか）
- **Inf**: 推論時間（ミリ秒、低いほど高速）
- **Device**: 
  - `NPU (QNN)` = NPUで実行中 ✅
  - `CPU` = CPUフォールバック（NPU利用不可）
- **緑の枠**: 検知された物体を囲む
- **ラベル**: `クラス名 信頼度`

## ⚡ パフォーマンスの目安

| デバイス | FPS | 推論時間 | 特徴 |
|---------|-----|----------|------|
| **NPU (QNN)** | 28-30 | 12-15ms | ⚡最速・省電力 |
| **CPU** | 8-10 | 100-120ms | フォールバック |

NPU使用時にFPSが30前後なら正常に動作しています！

## 🔧 トラブルシューティング

### ❌ QNNExecutionProviderが見つからない

**症状:**
```
⚠️ WARNING: QNN Execution Provider is NOT available.
```

**原因と対策:**

1. **onnxruntime-qnnがインストールされていない**
   ```powershell
   pip install onnxruntime-qnn
   ```

2. **通常のonnxruntimeと競合している**
   ```powershell
   pip uninstall onnxruntime onnxruntime-qnn
   pip install onnxruntime-qnn
   ```

3. **NPU非搭載デバイス**
   
   残念ながら、このデバイスではNPUが利用できません。CPUモードで動作します。

### ❌ モデルが見つからない

**症状:**
```
❌ モデルが見つかりません: models/yolov8n.onnx
```

**対策:**

`models/` ディレクトリに `yolov8n.onnx` を配置してください。

### ❌ カメラが開けない

**症状:**
```
❌ 入力を開けませんでした
```

**対策:**

1. 他のアプリ（Zoom、Teamsなど）がカメラを使用していないか確認
2. 別の入力ソース（画像・動画）を試す
3. デバイスマネージャーでカメラが認識されているか確認

### ❌ RTSPストリームに接続できない

**症状:**
```
❌ 入力を開けませんでした
```

**対策:**

1. `.env` ファイルが存在するか確認
2. RTSP_URLが正しいか確認
3. ネットワークカメラがオンラインか確認
4. ユーザー名・パスワードが正しいか確認
5. ファイアウォールで554ポートがブロックされていないか確認

### ⚠️ CPUで動作している

**症状:**
```
Device: CPU
```

画面に「CPU」と表示される場合、NPUが使用されていません。

**対策:**

1. `check_qnn.py` を実行して環境を確認
2. QNNExecutionProviderが利用可能か確認
3. NPU搭載デバイスか確認

## 💡 ヒント

- **初回起動は遅い**: QNNがモデルをコンパイル・最適化するため、初回起動は30秒程度かかることがあります
- **2回目以降は高速**: コンパイル済みモデルがキャッシュされるため、2回目以降は即座に起動します
- **推論時間をチェック**: 画面左上の「Inf: XXms」を見て、NPU使用時は12-15ms程度になっているか確認
- **明るい環境で**: 検知精度は照明条件に影響されます

## ❓ よくある質問

**Q: NPUとCPUで精度は変わりますか？**

A: 変わりません。推論速度だけが変わります。

**Q: カメラなしでも動作しますか？**

A: はい。画像ファイルや動画ファイルでも動作します。

**Q: 複数のカメラを同時に使えますか？**

A: 現在のバージョンでは1つの入力のみ対応しています。

**Q: 検知結果を保存できますか？**

A: 現在のバージョンでは画面表示のみです。保存機能が必要な場合はカスタマイズが必要です。
