# プロジェクト概要

NPU Object Detection Demoは、Qualcomm NPUを活用したシンプルな物体検知デモプロジェクトです。

## 🎯 目的

このプロジェクトの目的は、ONNX Runtime QNN Execution ProviderによるNPUアクセラレーションの使い方をシンプルに示すことです。

## 📁 ファイル構成

### **メインプログラム（2つ）**

#### `simple_demo.py`
- **役割**: YOLOv8による物体検知デモ
- **機能**:
  - Webカメラ、画像、動画、RTSPストリーム入力対応
  - ONNX Runtime QNN Execution Providerを使用したNPU推論
  - リアルタイム物体検知と可視化
- **依存**: `onnxruntime-qnn`, `opencv-python`, `numpy`, `python-dotenv`
- **実行方法**: `python simple_demo.py`

#### `check_qnn.py`
- **役割**: QNN環境の確認
- **機能**:
  - QNNExecutionProviderの利用可否チェック
  - セッション作成テスト
  - 環境情報の表示
- **依存**: `onnxruntime-qnn`
- **実行方法**: `python check_qnn.py`

### **セットアップスクリプト（2つ）**

#### `scripts/setup_windows_portable.ps1`
- **役割**: ポータブルPython環境のセットアップ
- **機能**:
  - Micromambaのダウンロード
  - Python 3.11環境の作成
  - 必要パッケージのインストール
  - QNN環境の確認
- **対象**: Pythonがインストールされていない環境
- **実行方法**: `.\scripts\setup_windows_portable.ps1`

#### `scripts/debug_snpe.ps1`
- **役割**: SNPE/QNN環境のデバッグ（オプション）
- **機能**:
  - ONNXモデルからSNPE DLC形式への変換
  - SNPE SDK環境の確認
  - デバッグ情報の出力
- **対象**: SNPE SDK利用時
- **実行方法**: `.\scripts\debug_snpe.ps1`

### **モデル**

#### `models/yolov8n.onnx`
- **モデル**: YOLOv8 Nano（ONNX形式）
- **入力**: 1x3x640x640 (NCHW, RGB, 0-1正規化)
- **出力**: 1x84x8400 (検出候補)
- **検知クラス**: COCO 80クラス

### **設定ファイル**

#### `.env`
- **役割**: 環境変数の設定
- **内容**:
  - `RTSP_URL`: RTSPストリームURL
  - `RTSP_USERNAME`: RTSP認証ユーザー名
  - `RTSP_PASSWORD`: RTSP認証パスワード

#### `.env.example`
- **役割**: `.env`のサンプル
- **使い方**: コピーして`.env`を作成

## 🔄 データフロー

### 1. セットアップフロー

```
setup_windows_portable.ps1
  ↓
Micromambaダウンロード
  ↓
Python 3.11環境作成
  ↓
パッケージインストール
  (onnxruntime-qnn, opencv-python, numpy, python-dotenv)
  ↓
QNN環境確認
  ↓
完了
```

### 2. 確認フロー

```
check_qnn.py
  ↓
ONNX Runtime読み込み
  ↓
Available Providers確認
  ↓
QNNExecutionProvider検出?
  ├─ Yes → テストセッション作成 → 成功
  └─ No → 警告表示
```

### 3. 推論フロー

```
simple_demo.py
  ↓
入力ソース選択
  (Webカメラ / 画像 / 動画 / RTSP)
  ↓
ONNX Runtimeセッション作成
  (QNNExecutionProvider優先)
  ↓
フレーム取得
  ↓
前処理
  (リサイズ, パディング, 正規化)
  ↓
NPU推論
  (YOLOv8モデル)
  ↓
後処理
  (信頼度フィルタ, NMS)
  ↓
描画・表示
  ↓
[ループ] → qキーで終了
```

## 🔑 主要技術

### ONNX Runtime QNN

- **パッケージ**: `onnxruntime-qnn`
- **役割**: ONNXモデルをQNN経由でNPU実行
- **特徴**: 
  - Qualcomm NPU（HTP）に最適化
  - 自動グラフ最適化
  - 低消費電力

### YOLOv8

- **モデル**: YOLOv8 Nano
- **用途**: リアルタイム物体検知
- **特徴**:
  - 軽量（約6MB）
  - 高速（NPU使用時12-15ms）
  - 80クラス対応

### OpenCV

- **パッケージ**: `opencv-python`
- **役割**: 画像・動画処理
- **用途**:
  - カメラ入力
  - 画像前処理
  - 結果描画

## 📊 パフォーマンス指標

### 推論速度

| 環境 | 推論時間 | FPS | 備考 |
|------|---------|-----|------|
| NPU (QNN) | 12-15ms | 28-30 | 省電力 |
| CPU | 100-120ms | 8-10 | フォールバック |

### リソース使用量

| 環境 | CPU使用率 | メモリ | 消費電力 |
|------|-----------|--------|----------|
| NPU | 10-15% | ~500MB | 低 |
| CPU | 80-90% | ~300MB | 高 |

## 🎯 設計原則

### 1. シンプルさ
- ファイル数を最小限に（実質4ファイル）
- 外部依存を最小限に（4パッケージのみ）
- 理解しやすいコード

### 2. 自己完結性
- すべての機能が`simple_demo.py`に含まれる
- モジュール分割なし
- 1ファイルで完結

### 3. 移植性
- ポータブル環境対応（micromamba）
- グローバルPython不要
- 環境汚染なし

def save_detection(frame, detections, output_dir="results"):
    timestamp = time.strftime("%Y%m%d_%H%M%S")

# 2. 検知イベントのロギング

    for det in detections:
        if det['class_name'] in target_classes:
            print(f"⚠️ Alert: {det['class_name']} detected!")
